tôi muốn bạn giúp tôi viết một đoạn code NodeJs để thực hiện các bước trích xuất thông tin từ file html theo hướng dẫn dưới đây:
1. cách trích xuất thông tin từ html:
    1.1 tìm tất cả thẻ có class "SetPageTermsList-term" trong file html.
    1.2 với mỗi thẻ tìm được, tìm thẻ con có data-testid là "set-page-term-card-side" 
        và trích xuất nội dung của thẻ đó.
    1.3 mỗi thẻ "SetPageTermsList-term" có 2 thẻ con có data-testid là "set-page-term-card-side", 
        một thẻ chứa Kanji và một thẻ chứa thông tin về định nghĩa. 
        Bạn cần trích xuất cả hai thông tin này và lưu vào file output theo định dạng JSON.
        "validSelector": {
            "parent":".SetPageTermsList-term",
            "child":"[data-testid=\"set-page-term-card-side\"]"
        }
    1.4 lưu kết quả vào file output theo định dạng JSON, với mỗi mục có cấu trúc như sau:
        {
            "kanji": "nội dung thẻ chứa Kanji",
            "definition": "nội dung thẻ chứa định nghĩa"
        }
        filename là tên của file html đã đọc, nhưng đổi phần mở rộng thành .json và lưu trong thư mục data\output.
    1.5 những file đã xử lý xong sẽ được lưu vào thư mục data\processed để tránh xử lý lại trong tương lai.

 2. Cách tổ chức thư mục:
    - input\groups.json: chứa danh sách các nhóm cần xử lý, mỗi nhóm có một file html tương ứng trong thư mục data\raw_html.
    - data\raw_html: chứa các file html cần xử lý.
    - data\output: nơi lưu kết quả sau khi trích xuất thông tin từ các file html.
    - data\processed: nơi lưu các file html đã được xử lý để tránh xử lý lại trong tương lai.
3. Cách thực hiện:
    3.1 Đọc file input\groups.json để lấy danh sách các nhóm cần
        xử lý.
    3.2 Với mỗi nhóm trong danh sách, tìm file html tương ứng trong thư
        mục data\raw_html.
    3.3 Đọc file html đó và trích xuất thông tin cần thiết theo các bước đã nêu ở phần 1.
    3.4 Lưu kết quả vào file output theo định dạng JSON.
    3.5 Di chuyển file html đã xử lý vào thư mục data\processed để tránh xử lý lại trong tương lai.
4. Lưu ý:
    - Đảm bảo rằng các file html trong thư mục data\raw_html có tên trùng với tên nhóm trong file input\groups.json để dễ dàng tìm kiếm và xử lý.
    - Sử dụng thư viện phù hợp để đọc và trích xuất thông tin từ file html.
    - Đảm bảo rằng quá trình trích xuất thông tin được thực hiện một cách chính xác và hiệu quả để tránh mất mát dữ liệu quan trọng.    
5. Kết quả cuối cùng:
    - Sau khi hoàn thành quá trình trích xuất thông tin, bạn sẽ có một file JSON trong thư mục data\output cho mỗi nhóm, chứa thông tin về Kanji và định nghĩa tương ứng. Các file html đã được xử lý sẽ được di chuyển vào thư mục data\processed để đảm bảo rằng chúng không bị xử lý lại trong tương lai. 
6. sử dụng code NodeJs.

tạo web app mobile offline(PWA) để hiển thị dữ liệu từ database NoSQL data\database\quizlet_db.json.
có thể chọn group từ header.
Để tạo một web app mobile offline (PWA) để hiển thị dữ liệu từ cơ sở dữ liệu NoSQL `data\database\quizlet_db.json`, 
chúng ta sẽ sử dụng Node.js React để xây dựng frontend. 

1. phần đọc file JSON và lưu vào database NoSQL, phải được tách riêng thành một module để có thể tái sử dụng 
   trong quá trình phát triển web app. cũng như dễ chuyển đổi sang các loại database khác trong tương lai nếu cần thiết.
2. không cần backend, chỉ cần frontend để đọc dữ liệu từ file JSON và hiển thị trên web app.
3. sử dụng React để xây dựng giao diện người dùng, với khả năng chọn nhóm từ header và hiển thị dữ liệu tương ứng.
4. sử dụng service worker để đảm bảo rằng web app có thể hoạt động offline và lưu trữ dữ liệu cục bộ.
5. đảm bảo rằng quá trình đọc file JSON và hiển thị dữ liệu được thực hiện một 
   cách hiệu quả để tránh mất mát dữ liệu quan trọng.
6. kết quả cuối cùng sẽ là một web app mobile offline (PWA) 
   có thể hiển thị dữ liệu từ cơ sở dữ liệu NoSQL `data\database\quizlet_db.json`, 
   với khả năng chọn nhóm từ header và hiển thị dữ liệu tương ứng.
7. Giao diện người dùng sẽ được thiết kế để dễ sử dụng trên thiết bị di động, 
   dễ sử dụng, và có khả năng hoạt động offline để đảm bảo rằng người dùng có 
   thể truy cập dữ liệu bất cứ khi nào họ cần.
8. chức năng chính là hiển thị theo từng group, chọn từng bài học để xem chi tiết về Kanji và định nghĩa tương ứng. 
   Giao diện sẽ được thiết kế để dễ dàng điều hướng và sử dụng trên thiết bị di động.
9. chức năng test cho mỗi bài học để người dùng có thể kiểm tra kiến thức của mình về Kanji và định nghĩa đã học. 
   cho từ kanji, và 4 options để người dùng chọn định nghĩa đúng.
   4 option đó sẽ được lấy ngẫu nhiên từ các định nghĩa khác trong cùng một group 
   để đảm bảo tính thử thách cho người dùng.
   Kết quả của bài test sẽ được lưu trữ cục bộ để người dùng có thể xem lại sau này.
10. chức năng học sẽ hiển thị dưới dạng card, mặt trước là Kanji, mặt sau là định nghĩa. 
    Người dùng có thể lật card để xem định nghĩa và có thể đánh dấu những card mà họ muốn học lại sau này.
    có thể đánh dấu đã thuộc, muốn học lại, hoặc chưa học để dễ dàng quản lý quá trình học tập của mình.
11. các chức năng là:
    - Hiển thị danh sách các nhóm (groups) từ file JSON.
    - Hiển thị chi tiết về Kanji và định nghĩa cho từng bài học trong nhóm đã chọn.
    - Chức năng test với 4 options để kiểm tra kiến thức của người dùng về Kanji và định nghĩa.
    - Chức năng học với card lật để hiển thị Kanji và định nghĩa, cùng với khả năng đánh dấu trạng thái học tập của người dùng.
    - Lưu trữ kết quả bài test và trạng thái học tập cục bộ để người dùng có thể xem lại sau này.
